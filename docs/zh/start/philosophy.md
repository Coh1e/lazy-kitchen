# 理念 / Philosophy

> 让人类拥有世界舌头！

懒蛋厨房不是菜谱网站。它是一个**结构化、AI 化、硬件极简、时间最优**的厨房操作系统。

## 五条核心信念

### 1. 内容是产品，网站只是查看皮肤

所有真知识在 `data/*.yaml`。网站 HTML 只是把 yaml 渲染成人类好读的样子。
你 clone 仓库 → 双击 `lazy-kitchen.html` → 离线、零依赖看完整手册。

### 2. AI 不在仓库里跑

仓库没有任何 LLM 客户端、没有 API key、没有服务器代码。
AI 在你本机的 CLI 里跑（Claude Code / Cline + DeepSeek / Codex / Gemini / Aider）。
仓库只放**指令** (`skills/add-dish/workflow.md`) 和**契约** (`schemas/*.json`)。

### 3. 硬件极简到 10 件

主厨刀 + 中式菜刀 + 炒锅 + 汤锅 + 量勺 + 厨房秤 + 真空封袋器 + 真空袋 + 茶包袋 + 冷冻标签。
**就这些。**
任何菜需要烤箱 / 电饭煲 / 空气炸锅 / 料理机，必须显式标注 `hardware_extension`。
能极简，是因为中国生鲜电商能买到切好的肉、剔骨的鸡腿。

### 4. 时间双轨：动手时间 vs 闲置时间

每道菜带 `hands_on_min` + `unattended_min`。
**懒的核心不是总时间短，而是动手时间短。**
4 小时炖牛腩（15 分钟动手）比 30 分钟全程紧张的快炒更"懒"。

### 5. 人是审美主导，AI 是劳动力

maintainer 用 add-dish skill 起草 → AI 跨家族二审 → 推到告示栏 → 社区投票评论 → 共识后正式发布。
不让 AI 决定品味，但用 AI 加速执行。

## 不做什么

- 不做账号系统（GitHub 账号即身份）
- 不做评论数据库（GitHub Discussions / Giscus）
- 不做后端（CF Pages 纯静态）
- 不做"网红营销"（让食物本身说话）
- 不做烘焙（v2 单独项目）
- 不做"低糖低脂版"标签（哪种饮食流派都不偏向）

## 不预制什么

- 基础食材（鸡蛋、豆腐、洋葱）—— 留人参与感
- 生鲜香草（香菜、薄荷、青柠）—— 当天用最香
- 切菜本身（生鲜电商代劳）

## 预制什么

- 干香料组合（DRY 包，常温保存数月）
- 湿酱腌料（WET 包，真空冷冻数月）
- 熟底（COOKED 块，冷冻几周）
- 汤底（SOUP 浓缩，冷冻数月）
- 即食热食（READY，备餐场景）

## 一句话

**Ingredient + Flavor SKU + SOP = Meal.**

让一周做饭从"30 个决策 × 7 天" 简化为"3 次大决策 + 5 分钟取出加热组装"。
